# trackpad-guard

## Manifesto

Software that claims authority over your computer is not your boss. A productivity app that disables your gestures, kills your terminal, and walls you into a single screen is making a bet: that if it bricks enough of your environment, compliance follows. This project is the counter-bet — that a user who understands their own system can preserve function without dismantling intent. We don't break the lock. We build a room the lock doesn't know about.

---

## Context

A productivity-boosting application under development exhibited the following behavior on macOS:

- Restricted the user to a single application window, preventing switching to other apps or desktops
- Disabled trackpad gestures (three-finger app switching, four-finger Mission Control)
- Opened Terminal and System Settings on launch, then immediately terminated both — cutting off in-session interference
- Maintained these restrictions for the duration of the session

The goal was not to disable the software or subvert its purpose. The goal was to preserve ambient system function — specifically trackpad gestures — and establish a clean parallel workspace outside the software's jurisdiction, without modifying the application itself.

---

## Architecture

Two independent layers were built:

### Layer 1 — Persistence (within the locked session)

A LaunchAgent runs continuously in the background, managed by `launchd` — macOS's process supervisor (PID 1). Because it is owned by `launchd` and not Terminal, the productivity app cannot kill it by targeting Terminal.app. Every 5 seconds, it reasserts the correct trackpad preference values. It also fires immediately on file change, providing a secondary trigger if the app overwrites the preference plists directly.

### Layer 2 — Escape Hatch (parallel session)

A second macOS user account (`Franz`) provides a completely isolated desktop environment. macOS Fast User Switching allows switching between sessions via a menu bar icon, without terminating the productivity app in the original session. A hot corner (bottom-right) triggers the lock screen, from which the user can switch to Franz. The productivity app, running under `austinkim`, has no reach into `Franz`'s session.

## Tooling

### `trackpad-guard.sh`

**Location:** `~/trackpad-guard.sh`
**Alias:** `tpg`

The distributable. A self-contained bash script that manages the full lifecycle of the persistence layer. Dynamically resolves all paths via `$HOME` — no hard-coded usernames. Drop it on any Mac, `chmod +x`, and run.

**Commands:**

| Command | Effect |
|---|---|
| `tpg install` | Writes the restore script and LaunchAgent plist, loads the agent |
| `tpg uninstall` | Unloads the agent, deletes all files — clean slate |
| `tpg enable` | Loads the LaunchAgent (agent must be installed) |
| `tpg disable` | Unloads the LaunchAgent without removing files |
| `tpg toggle` | Flips current state |
| `tpg status` | Reports RUNNING / INSTALLED+STOPPED / NOT INSTALLED |

---

### `restore-trackpad.sh`

**Location:** `~/Library/Scripts/restore-trackpad.sh`
**Generated by:** `tpg install`

The restore script. Writes four `defaults` keys across two preference domains — the built-in trackpad and Bluetooth trackpad — setting both three-finger horizontal swipe (app switching) and four-finger vertical swipe (Mission Control) to enabled (`2`). Includes a 2-second sleep at the top to beat the race condition where the productivity app writes its own values on exit after the LaunchAgent has already fired.

**Preference keys restored:**

```
com.apple.AppleMultitouchTrackpad
  TrackpadThreeFingerHorizSwipeGesture = 2
  TrackpadFourFingerVertSwipeGesture   = 2

com.apple.driver.AppleBluetoothMultitouch.trackpad
  TrackpadThreeFingerHorizSwipeGesture = 2
  TrackpadFourFingerVertSwipeGesture   = 2
```

---

### LaunchAgent — `com.user.trackpad-restore.plist`

**Location:** `~/Library/LaunchAgents/com.user.trackpad-restore.plist`
**Generated by:** `tpg install`

The persistence mechanism. A `launchd` job with two triggers:

- **`StartInterval: 5`** — fires every 5 seconds regardless of file state, handling API-level suppression during the productivity app's runtime
- **`WatchPaths`** — fires immediately when either trackpad plist is modified, catching direct file overwrites

Runs as the logged-in user. No elevated privileges. Survives Terminal being killed because it is managed by `launchd`, not a shell.

---

### Hot Corner — Bottom-Right → Lock Screen

**Configured via:** `defaults write com.apple.dock wvous-br-corner -int 13`

Moving the mouse to the bottom-right corner of the screen triggers the macOS lock screen. From there, the user can switch to the Franz session. Used because the productivity app blocks all keyboard shortcuts — hot corners are mouse-position events handled by the Dock process, below the application input layer.

---

### Fast User Switching — Franz

**Account:** username `Franz`, standard user (UID 502)
**Enabled via:** `defaults write com.apple.controlcenter "NSStatusItem Visible FastUserSwitching" -bool true`

A parallel macOS user session. The productivity app runs under `austinkim` (UID 501) and cannot reach into `Franz`'s session. Switching does not terminate the productivity app — it remains suspended in `austinkim`'s session, exactly as left. Franz's menu bar also has Fast User Switching enabled for returning.

**Limitations:**
- Clipboard is not shared between sessions
- Files are not shared unless explicitly placed in a world-readable location

---

### tmux

**Installed via:** `brew install tmux`

A terminal multiplexer. The productivity app kills Terminal.app on launch. tmux runs a server process independent of Terminal.app — if Terminal is killed, tmux sessions survive. Reattach with `tmux attach -t main` after reopening Terminal.

## Security Posture

| Property | Detail |
|---|---|
| Privilege level | User-space only — no `sudo`, no root, no kernel extensions |
| Footprint | Two files: a bash script and a LaunchAgent plist |
| Scope | Touches only trackpad preference domains |
| Reversibility | `tpg uninstall` removes all traces |
| Persistence | Survives reboots via `launchd`; loads at login |
| Distributability | Single script, fully portable via `$HOME` resolution |

The guard does not interfere with the productivity app's core function. It does not prevent the app from running, does not modify the app, and does not touch anything outside of trackpad preferences. The toggle (`tpg disable` / `tpg enable`) provides a clean off switch for sessions where the app's full intended behavior is desired.

---

## MO

Don't fight the lockdown inside its own session. Work around it by stepping into a parallel one. Let the app think it's in control — within `austinkim`'s session, it is. The persistence layer fights the specific collateral damage (gesture suppression) on a 5-second heartbeat. Franz is the clean room.

The principle: match the scope of resistance to the scope of the intrusion. The app restricts gestures and terminals. We restore gestures and preserve terminals. We don't touch the walls it actually built.
